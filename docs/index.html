Base API

A robust Node.js API starter template with TypeScript, following clean architecture principles and best practices for modern backend development.

View on GitHub Documentation

Features

This project is a seed for building a production-ready Node.js API with everything you need to get started quickly.

TypeScript

Type-safe code with modern JavaScript features for better developer experience and fewer bugs.

Clean Architecture

Clear separation of concerns with domain-driven design principles for maintainable code.

API Documentation

Auto-generated OpenAPI/Swagger documentation for easy API exploration and testing.

Authentication

Ready-to-use auth system with email verification and password reset functionality.

Database Integration

SQL support with TypeORM for efficient data access and management.

Testing Framework

Comprehensive test setup for unit, integration, and persistence tests.

Architecture

The project follows a clean, layered architecture with a clear separation of concerns:

Domain Layer

Located in /src/domain/

  • Entities: Core business objects like User and Organization
  • Migrations: Database schema migrations
  • Mappers: Object mapping service (e.g., Automapper or a custom mapping utility)

Application Layer

Located in /src/application/, organized by feature modules

  • Controllers: Handle HTTP requests/responses using TSOA decorators
  • Services: Contain business logic and orchestrate repository calls
  • Repositories: Handle data access, using TypeORM for CRUD operations
  • DTOs: Data Transfer Objects for input validation and response formatting

Infrastructure Layer

Located in /src/infrastructure/

  • Config:
    • authConfig: Authentication-related configs for better-auth
    • DbConnection: Database connection setup
    • Constants: Configuration constants
    • Server: Main server setup (Express + TSOA integration)
    • IoC Container: Dependency injection container setup using TSyringe
  • Utils: Shared utilities and adapters
    • Authentication: Express Authentication middleware
    • BetterAdapter: Integration with the better-auth authentication service
    • EmailAdapter: Integration with the email service
    • Mapper: Object mapping service
    • Logger: Logging utility
    • ErrorHandler: Central error handling middleware

Tests

Located in /tests/

  • Unit Tests: Isolate components, mock dependencies
  • Integration Tests: Test multiple layers together, often API endpoints
  • Persistence Tests: Focus on database interactions and entity mappings

Getting Started

Prerequisites

  • Node.js (v20.x or higher)
  • Yarn package manager
  • PostgreSQL database

Installation

git clone https://github.com/mravinale/base.api.git
cd base.api
yarn install

Environment Setup

mkdir -p src/infrastructure/config/env
cp .env.example src/infrastructure/config/env/.env.local

Start Development Server

yarn dev

Testing

The project includes a comprehensive test strategy with unit, integration, and persistence tests:

Unit Tests

Test individual components in isolation, mocking external dependencies to focus on business logic and edge cases.

Integration Tests

Test the interaction between multiple components to verify that different parts of the system work together correctly.

Persistence Tests

Focus on database interactions, verify entity mappings and relationships, and test complex queries and transactions.

Testing Tools

  • Mocha: Test runner
  • Supertest: For API testing
  • Chai: Assertion library
  • ts-mockito and sinon: Mocking libraries

Running Tests

yarn test                 # Run all tests
yarn test:file path/to/test  # Run specific test file
yarn test:coverage       # Run tests with coverage

CI/CD

The project uses GitHub Actions for continuous integration and deployment.

Workflow Features

  • Automatic builds and tests on pull requests and pushes to main
  • Code coverage reporting with Codecov
  • Security scanning with Snyk
  • PostgreSQL service container for integration tests
